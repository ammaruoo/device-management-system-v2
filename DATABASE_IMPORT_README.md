# دليل استيراد قاعدة البيانات

## المشكلة الأصلية
كان هناك خطأ في ملف `db.sql` الأصلي في تعريف view `laptop_status_report`:
```sql
CREATE TABLE `laptop_status_report` (
);
```
هذا التعريف فارغ ولا يحتوي على أعمدة، مما يسبب خطأ عند الاستيراد.

## الحل
تم إنشاء ملف `db_fixed.sql` مصحح يحتوي على:
- تعريفات صحيحة لجميع الجداول
- Views صحيحة بدون أخطاء
- Foreign Keys صحيحة
- **بيانات أساسية شاملة للبدء**

## كيفية الاستيراد

### 1. استخدام phpMyAdmin
1. افتح phpMyAdmin
2. أنشئ قاعدة بيانات جديدة
3. اذهب إلى تبويب "Import"
4. اختر ملف `db_fixed.sql`
5. اضغط "Go" لبدء الاستيراد

### 2. استخدام MySQL Command Line
```bash
mysql -u username -p database_name < db_fixed.sql
```

### 3. استخدام MySQL Workbench
1. افتح MySQL Workbench
2. اربط بقاعدة البيانات
3. اذهب إلى Server > Data Import
4. اختر ملف `db_fixed.sql`
5. اضغط "Start Import"

## محتويات قاعدة البيانات

### الجداول الأساسية
- **`branches`**: الفروع
- **`warehouses`**: المخازن
- **`warehouse_permissions`**: صلاحيات المستخدمين على المخازن
- **`warehouse_transfers`**: تحويلات المخازن
- **`warehouse_transfer_logs`**: سجل تحويلات المخازن
- **`operations`**: العمليات (محدث مع نظام تسجيل العمليات)
- **`broken_laptops`**: الأجهزة المعطلة
- **`users`**: المستخدمين
- **`notifications`**: الإشعارات

### Views
- **`laptop_status_report`**: تقرير حالة الأجهزة (مصحح)
- **`pending_transfers_summary`**: ملخص التحويلات المعلقة

## استكشاف الأخطاء

### إذا واجهت خطأ "Access denied"
```sql
GRANT ALL PRIVILEGES ON database_name.* TO 'username'@'localhost';
FLUSH PRIVILEGES;
```

### إذا واجهت خطأ في Character Set
```sql
ALTER DATABASE database_name CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### إذا واجهت خطأ في Foreign Keys
```sql
SET FOREIGN_KEY_CHECKS = 0;
-- استيراد البيانات
SET FOREIGN_KEY_CHECKS = 1;
```

### إذا واجهت خطأ في Views
```sql
-- إعادة إنشاء View
DROP VIEW IF EXISTS laptop_status_report;
CREATE VIEW laptop_status_report AS SELECT ...;
```
